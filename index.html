gym-system

gym247.up.railway.app

The problem

Small gyms run membership on spreadsheets and a shared notebook by the door — no record of who froze a membership, who's actually still active, or who's about to expire.

What it does

One admin dashboard to add, renew, freeze, and remove members. Each member gets a personal link with a QR code that acts as their entry pass — no app install, no card to lose.

The hard part

The QR code can't just be a static image of a member ID — anyone could screenshot it and hand it to a friend. I built it on TOTP (time-based one-time codes), the same idea behind authenticator apps, so the code embedded in the QR rotates on its own. A copied screenshot goes stale within the rotation window.

JavaScript Node.js Express React MongoDB TOTP / QR
Current limitation — the original goal is round-the-clock, unmanned entry. It's not there yet: that last step needs a QR scanner + door-lock controller on-site, which I don't have hardware for yet. The software side is ready for it.
