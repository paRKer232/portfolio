01

gym-system

https://gymsys-k71y.onrender.com

The problem

Small gyms run membership on spreadsheets and a shared notebook by the door — no record of who froze a membership, who's actually still active, or who's about to expire.

What it does

One admin dashboard to add, renew, freeze, and remove members. Each member gets a personal link with a QR code that acts as their entry pass — no app install, no card to lose.

The hard part

The QR code can't just be a static image of a member ID — anyone could screenshot it and hand it to a friend. I built it on TOTP (time-based one-time codes), the same idea behind authenticator apps, so the code embedded in the QR rotates on its own. A copied screenshot goes stale within the rotation window.

JavaScript Node.js Express React MongoDB TOTP / QR
Current limitation — the original goal is round-the-clock, unmanned entry. It's not there yet: that last step needs a QR scanner + door-lock controller on-site, which I don't have hardware for yet. The software side is ready for it.

02

QR Code Menu for Restaurants/Bars/Cafes

The problem

Small restaurants, bars, and cafés either reprint paper menus every time a price or item changes, or pay a third party monthly just to host a digital menu. Neither scales well for a place that tweaks its offering often.

What it does

Customizable menu dashboard where an owner adds, edits, and organizes menu items by category — no reprinting, no third-party subscription. Each table gets a QR code that opens the live menu instantly in the customer's browser, always showing whatever's currently in the dashboard.

The hard part

The menu had to update instantly everywhere the moment it's edited, without customers ever seeing a stale cached version or the owner needing to regenerate and reprint a QR code. I built the QR to point at a stable link tied to the venue rather than to a snapshot of the menu, so the dashboard is the single source of truth.

JavaScript Node.js Express MongoDB QR
